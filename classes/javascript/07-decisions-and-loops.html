Control Flow

  • There are certain programming constructs that determine the control flow (the order in which statements are executed) of your JavaScript code:

    1. The if statement and its different variations.

      if statement checks a condition, if it's true the code inside the if block will run. If condition is false, the code inside the block will be skipped.

      var num = 50;

      if (num > 10) { console.log("number is greater than 10"); // code inside the "if" block }

      // prints "number is greater than 10" to the console.

      var isItRaining = false;

      if (isItRaining) { console.log("bring an umbrella."); }

      // code inside "if" block is skipped.

      You can chain multiple else-if blocks after the if block in order to check for multiple conditions.

      var num = 12;

      if (num < 10) { console.log("number is less than 10"); } else if (num < 20) { console.log("number is less than 20"); } else if (num < 30) { console.log("number is less than 30"); }

      // prints "number is less than 20" to the console.

      And finally, you can add a single else block at the end of an if or the last else-if:

      var num = 100;

      if (num < 10) { console.log("number is less than 10"); } else if (num < 20) { console.log("number is less than 20"); } else if (num < 30) { console.log("number is less than 30"); } else { console.log("number is 30 or greater."); }

      // prints "number is 30 or greater" to the console.

      A conditional, or ternary, operator behaves similarly to an if-else statement.

      The syntax for it is:

      condition ? expression1 : expression2

      var isAbleToVote;

      // This if-else statement ...

      if (age < 18) { isAbleToVote = false; } else { isAbleToVote = true; }

      // ... can be written using the conditional operator ...

      isAbleToVote = age < 18 ? false : true;

    2. A switch statement takes a value and tries to match it to one of its case clauses, if a match is found the code associated with the matching case clause is executed :

      var numberOfWheels = 2;

      switch(numberOfWheels) { case 4: console.log("it has four wheels."); break; // If you don't add the keyword break at the end of each case's code, the code will fallthrough to the next case. We usually don't want that to happen! case 3: console.log("it has three wheels."); break; case 2: console.log("it has two wheels."); break; case 1: console.log("it has one wheel."); break; default: // Optional "default" clause code is run when no matches are found. console.log("Number of wheels is not in the range of 1 to 4."); }

      // prints "it has two wheels" to the console.

    3. Loops - a loop checks a condition and if the condition evaluates to true or truthy, then the code associated with the loop is executed. The loop code will repeat as long as the condition is true.

      • a for loop:

        var numbers = [100, 200, 300, 400];

        // A "for" loops has three parts wrapped inside a pair of parentheses. The parts are separated by a ;
        // 1st part: the initialization. This is where you would initialize a counter for the loop (e.g., var i = 0).
        // 2nd part: the condition. This is an expression (e.g., i < numbers.length) to be evaluated before the start of each loop. If the expression evaluates to true, then the loop code will run. If false, the loop exits.
        // 3rd part: the post expression. An expression to be evaluated after the loop code has ran. You use this part to write code that will eventually make the loop exit. (e.g., increment the counter: i+=1)

        for (var i = 0; i < numbers.length; i+=1) { console.log(numbers[i]); // will print contents of the "numbers" array using variable "i" as index for the array. }

      • a while loop:

        // A "while" loop runs as long as its condition (which is wrapped by a pair of parentheses) evaluates to true.

        var i = 0;

        while (i < 10) { console.log(i); // prints the value of i to the console during each run of the loop. i+=1; // make sure to write code that will lead to the end of the loop, or the loop might go on forever (NOT GOOD!). }

      • a do-while loop:

        // A "do-while" loop is similar to a "while" loop, the difference is a "do-while" loop checks its condition after loop code has ran. This means the loop code will run at least once.

        var i = 10;

        do { console.log(i); // prints a 10 to the console. } while (i != 10);

      • a for-in loop:

        // A "for-in" loop will iterate over all the enumerable properties of an object but in no particular order.

        var car = { doors: 4, wheels: 4, hasSunroof: false };

        // The following loop will repeat as many times as there are enumerable properties in the car object.
        // Each time the property variable will point to a different property of the car. for (var property in car) { console.log(property); // prints a different car property console.log(car[property]); // prints the value of the car's property to the console }

      • Use the keyword break to end a loop earlier than planned:

        var numbers = [100, 200, 300, 400];

        for (var i = 0; i < numbers.length; i+=1) { if (numbers[i] > 250) { break; // Stop the loop if current number in array is greater than 250. } console.log(numbers[i]); // will print contents of the "numbers" array using variable i as index for the array. }

      • Use the keyword continue to skip the current iteration of the loop and start the next iteration:

        var numbers = [100, 200, 300, 400];

        for (var i = 0; i < numbers.length; i+=1) { if (numbers[i] === 200) { continue; // Skip this iteration of the loop if current number in array is equal to 200. } console.log(numbers[i]); // will print contents of the "numbers" array using variable i as index for the array. }
