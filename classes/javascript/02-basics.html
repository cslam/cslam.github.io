JavaScript Basics

  • Data Types

    1. Number: 100, 0.5, 1e3, 0xffffff, etc.

    2. String: characters surrounded by single or double quotes: "15 dollars", 'some random text', etc.

    3. Boolean: true, false

    4. Object: {name: "Chris", age: 55}, [1, 2, 3], etc.

    5. Null: null

    6. Undefined: undefined

  • Variables

    • A variable stores a value of a certain data type which can then be retrieved by using the name of the variable.

    • To declare a variable use the keyword var followed by a name for the variable.

    • The name of the variable is called the identifier. A JavaScript identifier must start with a letter, underscore (_), or dollar sign ($); subsequent characters can contain the aforementioned characters along with digits (0-9).

    • Variable names are case-sensitive.

    • Declare a variable and give it a name called sum:

      var sum;

    • Store a value to a variable using the assignment symbol =:

      var sum = 100; // declare and assign a value to the sum variable.

      sum = 10; // assign a different value to the sum variable.

      var anotherSum = sum; // declare a new variable and assign sum's value to it.

    • The following two ways for declaring variables are the same:

      // Each variable separated by semicolon

      var i;

      var j;

      var k;

      // Each variable separated by comma with a semicolon after the final variable.

      var i, j , k;

    • You can assign a value to a variable without declaring var first on the variable name, BUT you should never do that. This will cause the variable to be accessible in the global scope. Always use var to declare your variables.

  • Operators

    • Assignment

      =: takes the expression (or value) to the right of the operator and stores it in the variable to the left of the operator.

    • Arithmetic operators:

      +, -, *, /, %: add, subtract, multiply, divide, and modulo (like divide, but returns the remainder only).

      +=, -=, *=, /=, %=: these are shorthand for 1) performing a mathematical operation on a variable and 2) assigning the result to that same variable.

      var num = 10;

      num += 50; // num = num + 50 which will make num to be 60

      ++, --: place it before or after a variable that is of a number type, and it will increment (++) or decrement (--) the number by 1 and then store that new value back to the variable. What's the significance of placing it before or after the variable?

      var num = 100;

      var anotherNum = ++num; // num and anotherNum are now 101

      var otherNum = num++; // otherNum is 101, num is now 102

      -: the negation operator is a unary operator because it takes only one operand. It returns the negation of the operand.

      var num = 100;

      var anotherNum = -num; // anotherNum contains the number -100

    • String operators:

      +: concatenates (combines) two strings.

      var greeting = "Hello" + " " + "Chris"; // final string: "Hello Chris"

      var numString = "100" + 2; // final string: "1002", the number turns to a string.

      +=: concatentates a string to a variable of string type and assigns the result to the variable.

      var greeting = "Hello ";

      greeting += "Chris"; // greeting now contains the string value "Hello Chris"

    • Comparison operators:

      <, <=, >, >=, ===, !==: less than, less than or equal, greater than, greater than or equal, strict equality, strict inequality.

      These operators take two operands, one on the left and another on the right side of the operator.

      The comparison operators will evaluate to a Boolean value of either true or false.

      === is the strict equality operator. It asks: is left hand operand equal to right hand operand?

      !== is the strict inequality operator. It asks: is left hand operand NOT equal to right hand operand?

      There are ==, and != equality and inequality operators, but in certain situations, they can give unexpected results. Use the safer versions, (=== and !==), instead.

      var x = 10 < 100; // variable x will contain the Boolean value of true

      var y = "10" < "2"; // strings are compared lexicographically (dictionary order). This example will evaluate to true, not false.

      "hello" === "hello"; // returns true

      100 !== 6; // returns true

    • Logical operators:

      • &&: logical AND accepts two boolean operands, if both operands are true then && returns true, else it returns false.

      • ||: logical OR accepts two boolean operands, if both operands are false then the || returns false, else it returns true.

        (10 < 100) && (10 > -5); // true AND true evaluates to true

        (10 < 100) && (10 < -5); // true AND false evaluates to false

        (10 < 100) || (10 < -5); // true OR false evaluates to true

        (10 > 100) || (10 < -5); // false OR false evaluates to false

      • !: logical NOT operator takes one operand only.

        The logical NOT operator can toggle a boolean value from true to false, and vice versa.

        var isItRaining = false;

        !isItRaining; // evaluates to true

    • Operator precendence

      • Operator precedence determines the order in which operators are evaluated. Operators with higher precedence are evaluated first.

        3 + 4 * 5; // evaluates to 23 because * has higher precendence than +

      • If you want an operator with lower precendence to operate first, then wrap that operator's operands with a set parentheses:

        (3 + 4) * 5; // evaluates to 35 because ( ) has higher precendence than *

      • Complete list of operator precendence can be found here: https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Operator_Precedence

    • There are other operators not mentioned here. You can check out the entire list of operators here: https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Expressions_and_Operators
